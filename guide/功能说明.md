# 键道6 Lua 扩展功能说明

适用于本配置（星猫键道6 · xmjd6）的 librime-lua 扩展功能。
所有功能在 macOS（鼠须管）/ Windows（小狼毫）/ Android（同文）/ iOS（自研 App）通用，
需要输入法内核带 librime-lua 插件。

---

## 一、功能速查表

### 1. 时间日期类（`lua/xmjd6/xmjd6_shijian.lua`）

| 输入 | 出什么 | 示例 |
|---|---|---|
| `rq` | 日期（横线式 / 中文式 / 大写式 / 农历+节气） | 2026-06-12、2026年06月12日、二〇二六年六月十二日 |
| `ej` | 时间、日期时间、Unix 时间戳 | 21:30:00、1781234567 |
| `xq` | 星期（中文 / 周数 / 英文） | 星期五、第23周 |
| `nl` | 农历、干支、时辰 | 丙午年四月廿七 |
| `jq` | 最近的节气及精确时刻 | 夏至 2026-06-21 ... |
| `=` + 8位日期 | 日历查询：公历⇉农历⇉干支⇉星期互转 | `=19910501` |
| `sjx/` | 纪念日/节日/节气倒计时，按剩余天数排序 | 距离端午节还有7天 |

`sjx` 是上游时间脚本传下来的触发码（"时间线"拼音首字母），`/` 结尾是为了与
`sjx` 码上的正常词（如"三角形"）区分。想改触发码见下文【配置方法】。

### 2. 倒计时配置（`anniversaries.txt`，放在 Rime 用户目录）

| 行格式（列间用 Tab） | 含义 |
|---|---|
| `名称 → 公历 → MMDD` | 公历纪念日，每年循环 |
| `名称 → 农历 → MMDD` | 农历纪念日，每年循环 |
| `名称 → 节气` | 二十四节气之一，公历日期每年天文推算 |
| 行首 `#` | 注释（禁用该行） |

- 候选按"最近要过的"升序排序；当天显示"今天是XX"。
- 文件没有任何有效行时显示内置示例。
- 除夕没有固定农历日期（腊月廿九或三十），看"春节"倒计时减一天。
- 写错的行自动跳过，不影响其他条目。

### 3. `=` 工具集（`xmjd6_tools.lua` + `number_tools.lua` + 计算器）

| 输入 | 出什么 | 示例 |
|---|---|---|
| `=?` | 功能帮助：列出需要输入码触发的功能候选 | `=uuid`、`sjx/`、`=add/词/编码;` |
| `=uuid` | UUID v4（小写/大写） | 550e8400-e29b-41d4-... |
| `=pw` / `=pw24` | 随机密码，默认16位可指定8~64位；候选分含符号/纯字母数字，已剔除易混淆的 0O1lI | |
| `=mem` | 当前 Lua 堆内存、已注册可释放缓存数 | Lua堆 2.31 MB |
| `=` + 10/13位数字 | Unix 时间戳（秒/毫秒）转日期时间，注释带几天前/后 | `=1718160000` → 2024-06-12 10:40:00 |
| `=` + 算式 | 计算器，支持 Lua 表达式、`\x.x^2\|` lambda、`$` 链式调用、`#` 代空格 | `=1+1`、`=floor(9^0.5)` |
| `=` + 普通数字 | 汉字大小写金额读法 | `=123` → 壹佰贰拾叁 |
| `=1234.56` / `=rmb1234.56` | 人民币金额、数字读法、千分位 | 壹仟贰佰叁拾肆元伍角陆分 |
| `=255>16` | 十进制转指定进制 | FF、0xFF |
| `=16:ff>10` | 指定源进制再转换 | 255 |
| `=5~3` | 非负整数按位异或 | 6、0x6、0b110 |
| `=10km>mi` | 长度、重量、数据容量、温度换算 | 6.213711922 mi |

按下第一个 `=` 时会先闪出符号候选（＝ 々 〃），继续输入即进入工具模式，属正常现象。

#### 空码数字和英文符号直接上屏（`direct_ascii.lua`）

没有任何输入编码时，数字以及指定英文符号会直接上屏，不需要再按空格。例如连续输入
`2.` 会立即得到 `2.`。存在候选时不会介入，数字仍用于选重，符号仍交给标点处理。

可在 `xmjd6.custom.yaml` 的 Patch 中快速增删符号：

```yaml
patch:
  direct_ascii/digits: true
  direct_ascii/symbols: ".,:!?"
```

- `digits: true/false`：是否允许空码数字直接上屏；
- `symbols`：允许直接上屏的英文符号字符串，例如 `".,:!?+-"`；
- 修改后需要重新部署。

建议不要加入 `;`、`'`、`/`、`=`、`\`，它们分别用于快捷符号、连打、加工面板、
工具命令和字符工具，加入后会产生按键冲突。

### 4. 词库模糊搜索（`dict_search_trigger.lua` + `dict_search.lua`）

| 操作 | 效果 |
|---|---|
| 打字出候选后按 `?` | 取当前首选词，在指定词库列表里全文模糊搜索，候选显示词+编码 |

- 只在当前输入码是纯字母键道码时触发（等价于 `^[a-z]+?`）。
  `=?` 会走功能帮助，不会触发词库模糊搜索。
- 搜索范围在 `dict_search_trigger.lua` 顶部的 `DICT_FILES` 列表里配置，
  加 `--` 注释即可停用某个词库，删掉注释即启用。当前启用 18 个（约 106 万词条）。
- 实现为**流式扫描**：每次按 `?` 现场逐块（256KB）读词库文件、边读边匹配、用完即扔，
  词条不常驻内存——iOS 键盘扩展零负担，词库开多少个都不增加内存。
- 代价是每次搜索都重读文件。实测（Mac、18 个词库 106 万词条）：

| 场景 | 表现 |
|---|---|
| 常见词（很快凑满 200 条提前终止） | 36~230 ms |
| 查不到的词（须扫完全部文件才能确认） | ~550 ms |
| 任何情况下的常驻内存 | ≈ 0 |

> iOS 设备 CPU/闪存稍慢，最坏情况约 1~2 秒。候选上限 200 条
> （`MAX_CANDIDATES`），凑满即停，所以词越常见返回越快。
> 历史方案备忘：缓存式（词条拼成大字符串常驻）查询快至 1~3ms，
> 但 106 万词条要常驻 20MB+ 内存，iOS 上会被系统杀，故弃用。

### 5. 动态自造词（`dynamic_phrase*.lua`）

自造词用于临时/长期补充主词库里没有的词，写入 `dynamic_phrases.txt`，添加或删除后**不用重新部署**即可生效。

| 输入 | 效果 | 例子 |
|---|---|---|
| `=add/词/编码;` | 直接添加指定词和编码 | `=add/王宥宥/wyy;` |
| `=add/编码;` | 把最近一次上屏内容加入该编码 | 先上屏“王宥宥”，再输入 `=add/wyy;` |
| `=add/N/编码;` | 把最近 N 段上屏内容拼起来加入该编码 | 先上屏“王”再上屏“宥宥”，输入 `=add/2/wyy;` |
| `=del/编码;` | 删除该编码下全部自造词 | `=del/wyy;` |
| `=del/词;` | 删除该词的全部自造编码 | `=del/王宥宥;` |
| `=del/词/编码;` | 精确删除某个“词+编码” | `=del/王宥宥/wyy;` |
| `=del/N/编码;` | 用最近 N 段上屏内容拼成词，再按编码精确删除 | `=del/2/wyy;` |

说明：

- 推荐在手机上用末尾 `;` 执行，例如 `=add/2/wyy;`。
- 桌面端也保留空格/回车确认，但 Android 上空格/发送键可能被宿主截走，`;` 最稳。
- `=add/2` 只是“最近 2 段”的半截命令，会提示继续输入 `/编码`，完整形式是 `=add/2/wyy;`。
- 自造词候选注释带 `〔自造〕`，同码时通常排在主词库候选前面。
- 多个自造词同码时，按 `dynamic_phrases.txt` 里的先后顺序显示。
- 重复添加同一个“词+编码”不会重复写入。
- 删除自造词时会同步清理 `candidate_order.txt` 里涉及该词的调频记录；如果该记录曾触发链式补码，也会清掉对应链式挪码记录，避免已删自造词被调频重新显示。


### 6. 动态调频 / 候选调频 / 挪码（`candidate_order.txt`）

用于把后面的候选提到前面的编码，同时把被挤下来的词按键道6规则顺势补码；改 `candidate_order.txt` 后不用重新部署即可生效。

| 文件/命令 | 效果 | 示例 |
|---|---|---|
| `0` | 打字时把第二候选/当前高亮候选提到首选，并写入 `candidate_order.txt` | 输入 `bmms`，按 `0` |
| `candidate_order.txt` | 运行时调频配置，一行一条 | `边骂<Tab>bmmsa<Tab>编码<Tab>bmms<Tab>bmmsa` |
| `coerr` | 显示 `candidate_order.txt` 解析错误 | 行格式错误时使用 |
| `python3 scripts/apply_candidate_order.py --dry-run` | 预览固化结果 | 不改文件 |
| `python3 scripts/apply_candidate_order.py --apply` | 生成 `xmjd6.candidate_order.dict.yaml` 并移除原词库旧项 | 执行后重新部署 |

格式：

```text
提到前面的词<Tab>它原来的码<Tab>被挤下来的词<Tab>目标码<Tab>被挤词补码
```

例：

```text
边骂	bmmsa	编码	bmms	bmmsa
词频	ckpbo	次品	ckpb	ckpbo
```

打字时用法：

1. 输入目标码，例如 `qzyw`。
2. 如果要把第二候选提到首选，直接按 `0`。
3. 如果要调第 3/4/5 候选，先把它高亮，再按 `0`。
   - 电脑：用方向键/候选移动键把候选高亮。
   - iOS / Android 自研键盘：**长按候选只高亮不提交**，再按 `0` 调频。
4. Lua 会自动把“第二候选/当前高亮候选、它原来的码、原首选、当前输入码、被挤词补码”写入 `candidate_order.txt`。
5. 如果首选已经是调频提上来的词，再对被挤下来的候选按 `0`，会撤销这条调频记录，不会追加反向同码记录。
6. 重新输入这个编码，即可看到新顺序。

示例：

```text
原来：qzyw -> 兆运，qzywv -> 赵云
操作：输入 qzyw，按 0 调第二候选“赵云”
结果：qzyw -> 赵云；原首选“兆运”仍在 qzyw 候选中，提示 ~u
```

手写配置效果：`bmms` 出“边骂”，原 `bmms` 的“编码”补到 `bmmsa`，在 `bmms` 候选里提示 `~a`；`ckpb` 出“词频”，原 `ckpb` 的“次品”补到 `ckpbo`，在 `ckpb` 候选里提示 `~o`。

同步说明：

- 日常调频只会改 `candidate_order.txt`，多端同步这一个 txt 即可。
- iOS、Android 和 Mac 小企鹅 WebDAV 同步入口会按自造词同目录规则，同时同步 `dynamic_phrases.txt` 和 `candidate_order.txt`。
- 第一次安装/更新 Lua、schema 后需要重新部署一次。
- 之后只修改或同步 `candidate_order.txt`，不用重新部署。
- 只有运行 `python3 scripts/apply_candidate_order.py --apply` 固化到 `xmjd6.candidate_order.dict.yaml` 或原词库后，才需要重新部署。

> 第一次安装/更新 Lua 和 schema 后仍需重新部署一次；之后只改 `candidate_order.txt` 不需要重新部署。详细说明见 `candidate_order_usage.md`。

性能与后续优化：

- 按 `0` 调频是低频操作，但为了给被挤词找“不重复”的补码，运行时需要判断候选补码是否已被词库占用。
- 当前实现已经从“建立全词库占用表”改成“只扫描本次可能补到的几个候选码”。这会明显降低按 `0` 时的 Lua 临时内存峰值：
  - 旧思路：临时建立约 40 多万个编码的占用表，峰值可到几十 MB 级。
  - 当前思路：只保存本次命中的候选补码占用，通常只有几条，函数结束后等待 GC 回收。
- 但当前方案仍然需要读取 `xmjd6.cizu`、`xmjd6.fjcy` 等较大的词库文件，所以第一次按 `0` 的体感卡顿可能仍然存在；这主要是文件 IO / 行扫描成本，不是长期常驻内存。
- 后续可继续优化为“分片占用码索引”：
  - 部署或打包前预生成类似 `candidate_order_index/ytyd.txt` 的小索引文件。
  - 按 `0` 时只读取当前目标码前缀对应的小文件，再叠加 `dynamic_phrases.txt` / `candidate_order.txt`。
  - 优点：显著减少首次按 `0` 的 IO，手机端体感更顺。
  - 代价：多一批派生索引文件；词库变化后要重新生成；打包/同步时不能漏。


### 7. 整句连打与文本加工（`sentence_mode` + `text_transform.lua`）

#### `'` 前缀整句连打：推荐方式

空码时先输入 `'` 进入整句模式，再连续输入整句话的键道编码。该模式不会顶功，
也不会因为中间编码暂时无候选而提前上屏：

- 可在方案选单中切换“连打开 / 连打关”，默认开启；
- 关闭后空码按 `'` 输出普通撇号，已有编码中的 `'` 仍作为分隔符；

- 输入 `'`：分隔一简、不能顶功或容易歧义的前后编码，不再执行三选；
- 有当前编码时按空格：在完整编码串末尾追加 `'` 分隔符，不清空之前的输入；
- 当前编码已经以 `'` 结尾时再按空格：提交当前整句候选；
- 达到普通输入的顶功条件时：自动插入 `'`，新按键作为下一段编码；
- 回车仍由 Rime 默认编辑器处理，用于上屏当前原始编码；
- 按 `/`：整句不上屏，直接打开包裹加工面板；
- 按退格：继续修改整句编码；
- 退格清空：退出整句模式。

例如：

```text
' + 整句话的连续编码 + 空格
```

如果最后按 `/`，可以选择：

- `“文本”`、`「文本」`、`『文本』`
- `《文本》`、`（文本）`、`【文本】`
- `[文本]`、`(文本)`
- `` `文本` ``、`**文本**`
- 英文小写、大写、`snake_case`、`kebab-case`、`camelCase`、`PascalCase`

例如在 `'` 整句模式中得到“重要通知”，按 `/` 后选择书名号候选，
宿主应用最终只收到：

```text
《重要通知》
```

不会先出现“重要通知”再重复插入一次。

#### 内容已经上屏

普通 librime-lua 只能记录上屏历史，无法把退格或撤销可靠地发送给微信、浏览器等宿主应用，
所以不能仅靠配置跨平台自动删除已经提交的原文。正确流程是：

1. 使用宿主应用的撤销键，让刚才的原文消失；
2. 输入 `=wrap`；
3. 从带“历史·请先撤销原文”注释的候选中选择加工结果。

桌面端使用应用自己的 `Cmd+Z` / `Ctrl+Z`；手机端可在自定义键盘上增加撤销按钮。

#### 合并最近几段

```text
=join/3
```

会把最近三段上屏内容按原顺序组成多种候选，例如：

```text
苹果、香蕉、西瓜
苹果，香蕉，西瓜
苹果, 香蕉, 西瓜
苹果 香蕉 西瓜
```

如果要用合并结果替换宿主里已经存在的三段文字，仍需先在宿主应用中撤销原文。

手机端不再必须增加 `Ctrl+G` 按钮：任何时候只要当前有候选，按 `/` 就会加工当前候选；
在 `'` 整句模式中则加工尚未上屏的完整句子。没有候选时 `/` 会继续交给原有命令处理。
`Ctrl+G` 仍作为同类功能的备用电脑热键。

热键可在 `xmjd6.custom.yaml` 中修改：

```yaml
patch:
  text_transform/hotkey: "Control+g"
  text_transform/sentence_hotkey: "slash"
  text_transform/sentence_prefix: "'"
```

## 二、配置到自己方案的方法

以下片段以 `<方案>.schema.yaml` 为准。`@*xmjd6/xxx` 表示加载
`lua/xmjd6/xxx.lua`，目录名跟着你的 lua 子目录走。

### 公共依赖：mem_cleaner（必装）

多数模块 `require("xmjd6.mem_cleaner")` 做统一内存释放，把
`lua/xmjd6/mem_cleaner.lua` 一起拷走即可，schema 无需声明。

iOS/安卓宿主可在键盘收起时向 Rime 发送 keycode `0xFFFFFF`（XK_VoidSymbol），
即可一键释放全部 Lua 缓存（词库 blob、反查库、emoji 字典、时间模块等）。
该 sentinel 由 dict_search_trigger 接收，不装词库搜索就收不到。

### 时间日期 + 倒计时

```yaml
engine:
  translators:
    - lua_translator@*xmjd6/xmjd6_shijian
recognizer:
  patterns:
    jsq: "^=[^;']+$"      # =19910501 日历查询要走 = 引导
    sjx: "^sjx/$"         # 没有这条 sjx/ 打不出来！
```

文件：`xmjd6_shijian.lua`（懒加载入口）+ `shijian_impl.lua`（实现）。
倒计时配置 `anniversaries.txt` 放 Rime 用户目录（或发行版共享目录）。

改触发码（例：sjx/ → djs/）需改三处：
1. schema 的 `sjx: "^sjx/$"` → `"^djs/$"`
2. `xmjd6_shijian.lua` 的 `TRIGGERS["sjx/"]` → `["djs/"]`
3. `shijian_impl.lua` 里 `input == "sjx/"` → `"djs/"`

### = 工具集 / 计算器 / 数字大写

```yaml
engine:
  translators:
    - lua_translator@*xmjd6/xmjd6_tools     # uuid/密码/时间戳/内存
```

若方案有顶功，顶功/空码回退处理器需放行 `=` `\` `&` 开头的输入
（本配置的 xmjd6_topup_processor.lua / auto_fallback.lua 已内置）。

### 词库模糊搜索（? 键）

```yaml
engine:
  processors:
    - lua_processor@*xmjd6/dict_search_trigger   # 越靠前越好，防止 ? 被截胡
  translators:
    - lua_translator@*xmjd6/dict_search
recognizer:
  patterns:
    dict_search: "^[a-z]+\\?$"
```

搜索哪些词库：编辑 `dict_search_trigger.lua` 顶部 `DICT_FILES`。
词库文件先在用户目录找，找不到回退共享目录（适配 iOS App Group）。

### 动态自造词（=add / =del）

```yaml
engine:
  processors:
    - lua_processor@*xmjd6/dynamic_phrase_processor  # 建议靠前，避免确认键被截胡
  translators:
    - lua_translator@*xmjd6/dynamic_phrase            # 建议放在主表 translator 前

dynamic_phrase:
  store_file: dynamic_phrases.txt

recognizer:
  patterns:
    dynamic_phrase: "^=(?:add|del)(?:/.*)?;?$"
```

文件：`dynamic_phrase_core.lua` + `dynamic_phrase.lua` + `dynamic_phrase_processor.lua`。

`dynamic_phrases.txt` 存在 Rime 用户目录，格式是一行一条：

```text
词<Tab>编码
```

注意：

- 第一次安装/更新 Lua 和 schema 后需要重新部署一次。
- 之后通过 `=add` / `=del` 改自造词，不需要重新部署。
- 如果宿主支持键盘收起清理缓存，发送 `0xFFFFFF` 给 Rime 后会释放自造词缓存；下次输入自造词编码时自动重载。
- Android 自研输入法里，`=add` / `=del` 命令的 `;`、空格、回车需要直通 Rime，不能在 Android 层直接选候选或提交原始输入。

### 动态调频 / 候选调频 / 挪码（candidate_order）

```yaml
engine:
  processors:
    - lua_processor@*xmjd6/candidate_order_processor   # 0 打字时一键调频
  translators:
    - lua_translator@*xmjd6/candidate_order            # 建议放在主表 translator 前
  filters:
    - lua_filter@*xmjd6/xmjd6_single_char            # 单字优先，先截断大候选流
    - lua_filter@*xmjd6/candidate_order_filter         # 建议放在单字优先后、uniquifier 前，避免 bq/te 这类大候选卡顿

candidate_order:
  enabled: true          # 总开关；false 时 0 不再调频，也不显示/隐藏 candidate_order 记录
  store_file: candidate_order.txt
  hotkey: "0"
```

文件：`candidate_order_core.lua` + `candidate_order.lua` + `candidate_order_filter.lua` + `candidate_order_processor.lua`。

`candidate_order.txt` 放 Rime 用户目录，格式：

```text
提到前面的词<Tab>它原来的码<Tab>被挤下来的词<Tab>目标码<Tab>被挤词补码
```

打字时：输入目标码 → 按 `0` 调第二候选；或先高亮想提前的候选再按 `0` → 重新输入同码看新顺序。手机端没有方向键时，在 iOS / Android 自研键盘里长按候选可移动高亮、不会提交候选，然后按 `0` 调频。被挤下来的原首选会继续显示在当前码下，并用 `~补码` 提示下一码，例如 `qzyw` 下显示 `兆运 ~u`。如果调完后首选已变成提上来的词，再对被挤下来的候选按 `0`，会撤销这条调频记录，避免生成反向同码记录。

动态调频总开关：默认开启，switcher 里会显示“调频开/调频关”。切到“调频关”后，`0` 不再调频，也不显示/隐藏 `candidate_order.txt` 里的调频记录。

如果想在配置里硬关，也可以在 `xmjd6.custom.yaml` 里写：

```yaml
patch:
  candidate_order/enabled: false
```

多端同步：日常只同步 `candidate_order.txt` 即可，不需要同步固化词库，也不需要重新部署；iOS、Android 和 Mac 小企鹅 WebDAV 同步入口会和自造词一样把它同步过去。要固化到词库时，在电脑上运行：

```bash
python3 scripts/apply_candidate_order.py --dry-run
python3 scripts/apply_candidate_order.py --apply
```

执行 `--apply` 后会更新 `xmjd6.candidate_order.dict.yaml`，并从原词库移除旧词条；之后重新部署。


### 文本加工、金额与单位转换

```yaml
engine:
  processors:
    - lua_processor@*xmjd6/text_transform
  segmentors:
    - affix_segmentor@sentence_mode
  translators:
    - table_translator@sentence_mode
    - lua_translator@*xmjd6/text_transform_translator
    - lua_translator@*xmjd6/number_tools

text_transform:
  hotkey: "Control+g"
  sentence_hotkey: "slash"
  sentence_prefix: "'"

sentence_mode:
  tag: sentence_mode
  dictionary: xmjd6.extended
  enable_sentence: true
  enable_user_dict: false
  prefix: "'"
```

`number_tools` 支持：

- 金额：`=1234.56` 或 `=rmb1234.56`
- 进制：`=255>16`、`=16:ff>10`，进制范围 2～36
- 异或：`=5~3`，仅支持非负整数，结果显示十进制、十六进制和二进制
- 长度：`mm/cm/m/km/in/ft/yd/mi`
- 重量：`mg/g/kg/oz/lb`
- 数据容量：`b/kb/mb/gb/tb`，按 1024 换算
- 温度：`c/f/k`

单位转换格式为 `=数值源单位>目标单位`，例如 `=10km>mi`、`=100c>f`、`=1024mb>gb`。
长度支持毫米、厘米、米、千米、英寸、英尺、码、英里；重量支持毫克、克、千克、盎司、磅；
数据容量使用二进制倍率（1 kb = 1024 b）；温度支持摄氏、华氏、开尔文。
不同类别的单位不会互相转换；无法识别的输入会留给原计算器处理。

## 三、内存机制速览（维护者参考）

| 模块 | 策略 |
|---|---|
| dict_search | 流式扫描零常驻：每次按 ? 逐块(256KB)读文件现场匹配，用完即扔 |
| xmjd6_shijian | 触发码命中才 require 实现（91KB 天文历法代码），_ENV 沙箱不污染全局 |
| lazy_simplifier | 开关打开才加载字典，关闭后空闲 30s 卸载 |
| for_hint / english | 反查库空闲 15s 卸载 |
| dynamic_phrase | 自造词首次命中后缓存 entries/by_code；=add/=del 后失效；注册到 mem_cleaner 后可在键盘收起时释放 |
| mem_cleaner | 上述全部缓存注册于此；宿主发 keycode 0xFFFFFF 一键全清 |

输入 `=mem` 可随时查看当前 Lua 堆与缓存数。
